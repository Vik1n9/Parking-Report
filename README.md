# 停車回報轉換 (Parking Report Converter)

此專案為一個用於處理和轉換特定停車場數據的網頁前端工具。主要功能是將自訂的簡化輸入格式，轉換為標準化的中控室語音回報講稿、可貼入 Excel 試算表的數據格式，以及保全可貼到 LINE 工作群組的手機回報文字。

## 使用入口

* **[中控收錄頁](https://vik1n9.github.io/Parking-Report/)**：`index.html`
  * 用於中控人員收錄無線電回報。
  * 產出中控口播文字與 Excel Tab 分隔值。
* **[保全手機回報頁](https://vik1n9.github.io/Parking-Report/guard.html)**：`guard.html`
  * 用於保全在無線電故障或需要直接回報時使用。
  * 以頁面內建數字鍵盤輸入，不依賴手機原生鍵盤。
  * 提供「分享到 LINE」與「複製 LINE 回報」按鈕，由保全在 LINE 內選擇群組並送出。

## 系統功能

* **多重輸入模式**：支援透過空格分隔的「文字輸入」以及切換焦點的「快速欄位輸入」。
* **自定義語法解析**：透過特定的數字與字母組合（如 `0.1~0.9`、`x`、`A~E`）來代表不同的車位狀態與區域。
* **自動產出報表**：
    * 顯示各區停車狀況。
    * 自動生成中控室口播用的文字稿。
    * 生成 Tab 分隔的數值，以供貼入假日停車場 Excel 報表。
* **剪貼簿整合**：提供「複製中控回報」與「複製 Excel 值」的快捷操作。
* **保全 LINE 回報**：提供手機版 `0~9` 數字鍵盤、台車模式、固定順序錄入與 LINE 分享連結。
* **保全計算機式輸入**：保全頁在數字鍵盤上方顯示目前格子的輸入值，並提供清除本格、上一格、下一格與刪除回退一碼。

## 輸入規則與欄位對照

系統預設解析的欄位順序固定為：
1. 一樓以上
2. 一樓以下
3. P1
4. P3
5. 紡A
6. 紡B
7. 紡C
8. 紡D
9. 紡E
10. 柏油路

### 數值與符號定義

欄位類型分兩種，輸入意義不同：
* **一樓以上、一樓以下**（IS_CAR 區域）：整數代表**剩餘車位數**。
* **P1、P3、紡A～E、柏油路**（非 IS_CAR 區域）：整數代表**成數**（例如 `8` 代表尚有 8 成）。

| 輸入值 | 意義 |
|---|---|
| 整數（如 `423`、`8`）| IS_CAR 區域：剩餘車位數；非 IS_CAR 區域：成數 |
| 小數 `0.1`～`0.9` | 小數點後一位即台車數，例如 `0.4` = 4 台車 |
| `0` | 車位全滿 |
| `x` | 該區域本次未啟用（排除於報表外） |

### 區域簡稱對照
* **`A` ~ `E`**：對應 紡A 至 紡E。
* **`R`**：對應 柏油路。

**輸入範例：**
字串 `423 256 7 7 E=0.4`

解析結果：

| 欄位 | 值 | 來源 |
|---|---|---|
| 一樓以上 | 423 | 第 1 個位置 token |
| 一樓以下 | 256 | 第 2 個位置 token |
| P1 | 7（成） | 第 3 個位置 token |
| P3 | 7（成） | 第 4 個位置 token |
| 紡E | 4 台車 | 具名指定 `E=0.4` |
| 紡A、紡B、紡C、紡D、柏油路 | `x` | 未提及，預設值 |

## 系統限制與潛在問題分析

檢視此網頁工具的設計與邏輯，存在以下客觀的設計缺陷與限制：

1.  **高度耦合的業務邏輯**：系統內的區域名稱（如「紡A」、「P1」）與欄位順序均為寫死（Hardcoded）的狀態。若現場停車場結構改變，或試圖將此工具轉移至其他廠區使用，必須直接修改原始碼，完全缺乏擴充性與模組化能力。
2.  **輸入語法違反直覺**：使用 `0.1~0.9` 來代表 `1~9` 台車的設計邏輯不符合一般使用者的認知習慣。這種為了規避解析器衝突而採取的取巧作法，提高了交接給其他人員時的學習成本。
3.  **錯誤處理機制匱乏**：缺乏嚴謹的防呆機制。若使用者輸入不符合既定格式的字元，或超出邏輯範圍的數值（如 `1.5` 或非預期的英文字母），系統可能無法正確攔截或解析，且介面未見明顯的錯誤驗證提示（Validation UI）。
4.  **格式相容性風險**：依賴特定字元（Tab）來貼齊 Excel 欄位，這在不同的作業系統或不同版本的試算表軟體（如 Google Sheets、舊版 Office、LibreOffice）中，貼上時可能會出現跑版、格式錯位或型別錯誤的情況。
5.  **有限的資料持久化**：系統透過瀏覽器 `localStorage` 自動儲存最近 40 筆回報紀錄，可於「歷史調閱」區塊查閱與重新載入。然而，一旦使用者手動清除瀏覽器儲存空間，或於無痕模式下使用，所有記錄將無法保留，亦無法跨裝置同步。
6.  **維護性低**：若未來需要增加新的車區（例如 P2、P4），現有的簡稱解析邏輯與陣列順序必須全面重構，極易引發連鎖錯誤。
